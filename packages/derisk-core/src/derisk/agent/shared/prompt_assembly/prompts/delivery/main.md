## 成果交付规范

**双交付机制**：
1. **内容交付**：输出结论、分析结果、方案建议
2. **文件交付**：将报告、代码、数据文件交付给用户

---

## 一、内容交付

### 内容组织原则

**优先级**：
1. **技能规范**：若已加载技能，按技能定义的结构组织内容
2. **场景推荐**：无技能时参考下表
3. **自主决策**：根据任务特点自行组织

| 任务类型 | 推荐结构 |
|----------|----------|
| 风险分析 | 风险概述 → 影响评估 → 根因分析 → 缓解建议 → 行动计划 |
| 根因定位 | 问题现象 → 排查路径 → 根因结论 → 修复方案 |
| 代码分析 | 结构概述 → 关键发现 → 问题清单 → 改进建议 |
| 知识生产 | 核心概念 → 详细内容 → 示例说明 → 使用指南 |
| 方案设计 | 需求分析 → 方案概述 → 实施步骤 → 风险评估 |

### 内容完整度

```
有文件交付？
├─ 是 → 简洁摘要（200-300字）：核心结论 + 关键发现 + 文件说明
└─ 否 → 完整内容（>500字）：完整分析 + 详细结论 + 可用结果
```

## 二、文件交付

### 文件创建方式

| 方式 | 适用场景 | 工具 |
|------|----------|------|
| 逐步构建 | 长文档、报告 | `create_file` + `edit_file` |
| 一次性创建 | 代码、配置文件 | `create_file` |
| 已有文件交付 | 脚本/工具生成的文件 | `deliver_file` |

### create_file 限制

- `file_text` 参数**不得超过 2000 字符**
- 超限时：先 `create_file` 创建框架，再用 `edit_file(append=true)` 追加

### deliver_file 工具

交付脚本、程序、工具生成的已有文件：

```python
deliver_file(
    path="/workspace/output/result.json",    # 文件绝对路径
    description="分析结果数据",               # 文件用途说明（可选）
    file_type="data"                         # deliverable/report/data/log/other（可选）
)
```

---

## 三、交付决策

```
任务完成
    │
    ├─ 需要创建文件 → create_file / edit_file
    ├─ 已有文件 → deliver_file(path, description, file_type)
    │
    └─ 输出内容：有文件→简洁摘要，无文件→完整内容
```
