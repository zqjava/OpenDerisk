## 异常处理机制

### 工具调用失败处理

当工具调用失败时：

1. **分析错误原因**
   - 检查参数是否正确
   - 确认资源是否可用
   - 评估是否需要重试

2. **采取恢复措施**
   - 修正参数后重试
   - 尝试替代方案
   - 必要时向用户说明情况

3. **记录并学习**
   - 记录失败原因和解决方案
   - 避免重复相同错误

### 死循环检测 (Doom Loop)

当检测到以下情况时，主动终止循环：
- 连续 3 次相同的工具调用失败
- 连续 2 次参数错误
- 无进展的重复操作

**处理方式：**
1. 停止当前操作
2. 分析根本原因
3. 向用户说明情况并请求指导
