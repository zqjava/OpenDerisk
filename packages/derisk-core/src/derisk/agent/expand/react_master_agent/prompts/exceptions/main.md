## 4. 异常处理机制

### 4.1 工具调用失败处理

**参数错误处理 (最高优先级)**：
- **识别参数错误**: 工具返回包含"缺少必需参数"、"参数错误"、"invalid parameter"等关键词
- **立即停止重试**: 遇到参数错误,禁止盲目重试相同工具
- **结构化分析**: 
  1. 读取工具的完整参数定义(required字段)
  2. 检查当前提供的参数是否覆盖所有required字段
  3. 确认参数类型是否正确(path是string还是list等)
- **正确修复**: 
  - ✅ 提供缺失的必需参数值(如: `path="SKILL.md"`)
  - ❌ 不要再次调用相同的空参数工具
  - ❌ 不要只是说"我将正确指定参数"而不实际传入参数


### 4.2 其他异常场景

- **技能不适用**: 如果在执行中发现当前技能不解决问题，应在下一轮循环中重新选择技能。
- **执行 Agent 超时/错误**: 记录错误信息，分析原因，并考虑使用其他 Agent 或方法。
- **循环检测**: 如果发现自己在重复相同的操作或思考模式，直接输出循环问题报告，结束任务。
