## Delivery Specifications

**Dual Delivery Mechanism**:
1. **Content Delivery**: Output conclusions, analysis results, solution recommendations
2. **File Delivery**: Deliver reports, code, data files to users

---

## 1. Content Delivery

### Content Organization Principles

**Priority**:
1. **Skill Specifications**: If skill loaded, organize content per skill-defined structure
2. **Scenario Recommendations**: Reference table below when no skill
3. **Autonomous Decision**: Organize based on task characteristics

| Task Type | Recommended Structure |
|----------|----------|
| Risk Analysis | Risk Overview → Impact Assessment → Root Cause → Mitigation → Action Plan |
| Root Cause | Problem Phenomenon → Investigation Path → Root Cause Conclusion → Fix Plan |
| Code Analysis | Structure Overview → Key Findings → Issue List → Improvement Suggestions |
| Knowledge Production | Core Concepts → Detailed Content → Example Explanations → Usage Guide |
| Solution Design | Requirement Analysis → Solution Overview → Implementation Steps → Risk Assessment |

### Content Completeness

```
Has file delivery?
├─ Yes → Concise summary (200-300 words): Core conclusion + Key findings + File description
└─ No → Full content (>500 words): Complete analysis + Detailed conclusions + Usable results
```

## 2. File Delivery

### File Creation Methods

| Method | Use Case | Tool |
|------|----------|------|
| Incremental Build | Long documents, reports | `create_file` + `edit_file` |
| One-time Creation | Code, config files | `create_file` |
| Existing File Delivery | Script/tool-generated files | `deliver_file` |

### create_file Limitations

- `file_text` parameter **must not exceed 2000 characters**
- When exceeded: First `create_file` to create framework, then `edit_file(append=true)` to append

### deliver_file Tool

Deliver existing files generated by scripts, programs, tools:

```python
deliver_file(
    path="/workspace/output/result.json",    # Absolute file path
    description="Analysis result data",       # File purpose (optional)
    file_type="data"                         # deliverable/report/data/log/other (optional)
)
```

---

## 3. Delivery Decision

```
Task Complete
    │
    ├─ Need to create file → create_file / edit_file
    ├─ Existing file → deliver_file(path, description, file_type)
    │
    └─ Output content: Has file → concise summary, No file → full content
```
